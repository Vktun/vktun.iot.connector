# 层间交互优化清单标注执行优先级与负责模块注意事项

本注意事项针对工业设备数据采集SDK层间交互逻辑优化精简执行清单，专门用于**标注执行优先级、匹配对应负责模块**环节，适配方案撰写、开发排期、落地管控全流程，兼顾方案规范性、开发可行性与工业场景特殊性，全程贴合SDK五层分层架构，避免标注混乱、权责错位、优先级不合理问题，具体分为通用基础注意事项、优先级标注注意事项、负责模块匹配注意事项、方案呈现注意事项四大类。

# 一、通用基础注意事项

- **严格贴合原有架构，不新增、不篡改模块**：必须匹配文档既定的五层架构（应用接口层、业务逻辑层、并发调度层、通信适配层、底层驱动层），禁止凭空新增模块、更改模块名称，确保标注内容与整体方案架构完全统一，无前后矛盾。

- **权责唯一对应，杜绝多模块模糊归属**：单条优化措施仅标注**核心负责模块**，可补充协同配合模块，严禁一条措施对应多个核心模块，避免开发阶段出现推诿、无人负责的情况，核心模块为直接执行落地的模块，协同模块为辅助配合模块。

- **优先级与工业场景强绑定**：优先级划分必须贴合工业SDK**高并发、高稳定、7×24h运行、弱网适配**核心诉求，优先保障稳定性、架构合规性，再兼顾性能与运维，不单纯以开发难度定优先级。

- **禁止跨层级乱标负责模块**：优化措施归属模块，需贴合措施执行逻辑，仅允许标注措施实际运行、开发对应的层级，严禁将底层优化标注给上层接口层，上层规范类措施不强行归属底层驱动模块。

- **标注语言简洁统一，不冗余**：优先级仅用高、中高、中、低四级标注，负责模块直接用层级简称，不额外添加冗长描述，保持清单简洁易读，适配方案查阅与开发排期场景。

# 二、执行优先级标注专项注意事项

**优先级划分核心原则**：先保架构合规与系统稳定，再提交互性能，最后补运维监控，贴合工业现场“零宕机、稳运行”的核心需求，严禁颠倒优先级顺序。

- **“高优先级”标注严控范围**：仅标注**架构刚性规范、防崩溃、防耦合**类措施，比如固化接口契约、拦截违规调用、全链路异步化，这类措施直接决定架构是否合规、SDK是否稳定运行，必须首批落地，不可延后。

- **禁止将性能优化标为最高优先级**：无锁队列、批量聚合等性能类措施，统一标为“中高”，不得凌驾于规范类、容错类措施之上，工业场景优先保证架构合规与运行稳定，而非极致性能。

- **“中优先级”标注边界清晰**：仅用于容错兜底、状态保障类措施，比如分级超时、熔断降级、状态校验，这类措施用于提升稳定性，需在核心规范落地后推进。

- **“低优先级”仅限非核心运维措施**：仅用于动态参数调优、资源联动管控等运维辅助类措施，可在SDK核心功能开发完成、稳定运行后补充落地，不影响核心交互逻辑。

- **禁止全量标高优先级**：需做梯度划分，高优先级控制在3条以内，中高优先级4-5条，中优先级3-4条，低优先级2条左右，避免优先级失效，无法指导开发排期。

- **优先级不随开发难度调整**：即便高优先级措施开发难度偏高，也不得降级标注，工业SDK架构合规性与稳定性，优先级高于开发效率。

# 三、负责模块匹配专项注意事项

- **规范类措施统一归属顶层模块**：接口契约、违规调用拦截、统一DTO等规范类措施，核心负责模块为**应用接口层+架构设计组**，协同模块为业务逻辑层，不归属底层通信、驱动模块，这类措施属于顶层架构管控范畴。

- **性能调度类措施归属核心调度层**：无锁队列、异步化、批量聚合、线程池管控等性能优化措施，核心负责模块为**并发调度层**，协同业务逻辑层、通信适配层，贴合并发调度层高并发管控的核心职责。

- **容错与状态类措施归属核心业务层**：超时管控、熔断降级、数据补发、状态校验等措施，核心负责模块为**业务逻辑层**，协同并发调度层、通信适配层，由业务中枢统筹全局状态与容错逻辑。

- **底层通信适配类措施归属对应插件层**：链路保活、协议解析、硬件交互相关优化，核心负责模块为**通信适配层**，底层驱动相关优化，归属**底层驱动层**，不跨层归属。

- **运维监控类措施为全局协同模块**：监控埋点、动态调优、资源管控类措施，核心负责模块为**业务逻辑层+并发调度层**，全局协同各层级，不单独归属某一层。

- **杜绝模块归属悬空**：每条措施必须标注核心负责模块，严禁出现“无模块负责”“全局负责”等模糊表述，明确开发对接主体，方便落地分工。

- **区分开发负责与运行归属**：标注模块为开发落地负责模块，而非运行时执行模块，避免开发分工与运行逻辑混淆，确保开发团队权责清晰。

# 四、方案呈现与落地配套注意事项

- **标注内容嵌入清单，不额外堆砌**：优先级与负责模块直接跟在每条优化措施后，采用简洁格式标注，不破坏原有清单结构，保持全文排版整齐、阅读流畅，不新增冗余段落。

- **同步标注说明，避免歧义**：清单顶部补充一句简短标注说明，明确优先级定义、模块简称含义，方便方案阅读者、开发人员快速理解，无需反复查阅前文。

- **不与原有注意事项冲突**：标注内容需贴合前文层间交互注意事项，比如严禁跨层调用的措施，负责模块不得标注跨层模块，保持全文逻辑一致。

- **适配工业现场落地场景**：优先级与模块分工，需考虑工业现场调试、运维实际情况，核心稳定类措施优先落地，便于现场快速部署、少出故障。

- **可追溯、可调整**：标注内容预留微调空间，可根据实际项目人力、工期小幅调整，但核心优先级与模块归属不得大幅改动，确保方案严谨性与可行性平衡。
