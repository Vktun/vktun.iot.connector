# 新增设备接入流程

本文档给出从新增设备到完成接入的标准流程，适用于真实设备、模拟设备和二次开发项目。

## 1. 确认设备接入方式

收集以下信息：

- 通信方式：TCP Client、TCP Server、UDP、Serial、HTTP、MQTT。
- 协议类型：Modbus RTU/TCP、S7、IEC104、自定义协议、HTTP、MQTT。
- 连接参数：IP、端口、串口、波特率、超时、TLS、认证。
- 点位清单：点位名称、地址、数据类型、单位、倍率、范围。
- 样例报文：至少一组正常请求/响应和一组异常响应。

## 2. 选择或创建协议模板

- 已有模板：复制并修改 `src/Vktun.IoT.Connector.Protocol/Templates/` 或 `templates/` 下的模板。
- 新协议模板：按 `协议模板字段说明.md` 补齐公共字段和点位字段。
- 自定义协议：明确帧头、长度、校验、偏移和数据类型。
- HTTP/MQTT：连接字段放到 `DeviceInfo.ExtendedProperties`，payload 解析规则单独说明。

## 3. 配置设备

最小设备配置应包含：

```csharp
var device = new DeviceInfo
{
    DeviceId = "device-001",
    DeviceName = "生产线设备 001",
    CommunicationType = CommunicationType.Tcp,
    ConnectionMode = ConnectionMode.Client,
    IpAddress = "192.168.1.10",
    Port = 502,
    ProtocolType = ProtocolType.ModbusTcp,
    ProtocolId = "modbus-template-001",
    ProtocolVersion = "1.0.0"
};
```

## 4. 本地验证

优先使用 DeviceMock 或本机服务验证：

```bash
dotnet run --project demo/Vktun.IoT.Connector.DeviceMock
dotnet test tests/Vktun.IoT.Connector.ProtocolTests
dotnet test tests/Vktun.IoT.Connector.UnitTests
```

验证内容：

- 能连接设备。
- 能发送请求。
- 能收到响应。
- 解析出的 `DeviceData.DataItems` 与点位清单一致。
- 断线、超时、异常码能产生明确错误。

## 5. 接入运行时

应用侧推荐使用主包 DI 注册：

```csharp
var services = new ServiceCollection();
services.AddVktunIoTConnector();

await using var provider = services.BuildServiceProvider();
var collector = provider.GetRequiredService<IIoTDataCollector>();

await collector.InitializeAsync();
await collector.AddDeviceAsync(device);
await collector.ConnectDeviceAsync(device.DeviceId);
var data = await collector.CollectDataAsync(device.DeviceId);
```

## 6. 现场联调检查

- 网络连通性：端口、路由、防火墙、TLS 证书。
- 设备并发限制：最大连接数、请求频率、会话保持。
- 超时配置：连接超时、请求超时、重试退避。
- 点位正确性：地址、字节序、字序、倍率、单位。
- 质量判断：异常码、越界值、校验失败、断线数据。

## 7. 完成标准

设备接入完成应满足：

- 模板已提交并通过校验。
- 至少有一组样例报文或 DeviceMock 回归。
- README 或接入文档说明了连接字段。
- 未实现能力已在 `实现状态与桩功能说明.md` 标记。
- 构建和测试通过。
