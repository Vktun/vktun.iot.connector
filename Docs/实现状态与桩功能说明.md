# 实现状态与桩功能说明

本文档用于标记当前仓库中仍未完全产品化、或刻意保留为替代入口的功能，避免使用者把演示代码、测试桩或受限实现误认为生产能力。

## 总体状态

| 模块 | 当前状态 | 使用建议 |
| --- | --- | --- |
| TCP/UDP/Serial 通道 | 已接入真实收发链路 | 可作为二次开发基础，真实设备联调仍需按现场协议验证 |
| HTTP Client 通道 | 已接入 `IHttpClientFactory` 与请求响应事件 | 适合 HTTP 设备/API 采集，按设备配置 `Url` 或 `BaseUrl + Path` |
| MQTT 通道 | 已接入 MQTT broker 发布/订阅链路 | 生产环境不要启用 `UseInMemoryTransport` |
| Modbus RTU/TCP 解析 | 已实现解析与命令打包 | 推荐优先用于真实设备回归 |
| S7 解析 | 已实现专用读写命令构建入口 | 不要直接调用通用 `BuildRequest` 桩入口 |
| IEC104 解析 | 已实现解析入口，通用命令打包受限 | 指令下发前需补齐具体 ASDU 命令模型 |
| DeviceMock | 用于开发和回归的设备模拟器 | 不是生产网关服务 |

## 明确标记的受限实现

| 位置 | 当前行为 | 说明 |
| --- | --- | --- |
| `AuthenticationService.ValidateCertificateAsync` | 证书认证为占位说明 | TLS 证书链验证应在 TLS 握手或专用证书验证服务中完成 |
| `CertificateManager` CRL 检查 | 存在 TODO | 当前不执行证书吊销列表检查，生产环境需接入企业 CA/CRL/OCSP 策略 |
| `ModbusRtuParser.BuildRequest` | 抛出 `NotImplementedException` 并提示使用 `PackCommand` | 这是故意保留的兼容入口，不是缺少 Modbus 命令能力 |
| `ModbusTcpParser.BuildRequest` | 抛出 `NotImplementedException` 并提示使用 `PackCommand` | 真实命令打包走 `PackCommand` |
| `S7ProtocolParser.BuildRequest` | 抛出 `NotImplementedException` 并提示专用方法 | S7 命令应使用 `BuildReadCommand` 或 `BuildWriteCommand` |
| `IEC104ProtocolParser.BuildRequest` | 抛出 `NotImplementedException` 并提示使用 `PackCommand` | IEC104 指令模型尚需按业务场景补齐 |
| Demo WPF Converter 的 `ConvertBack` | 抛出 `NotImplementedException` | UI 单向绑定场景使用，不影响采集 SDK |
| 测试项目中的 `TestConfigurationProvider` | 多个模板方法抛出 `NotImplementedException` | 仅用于单元测试最小依赖，不是运行时代码 |

## 发布前检查规则

- README 中只宣传已经有真实实现或明确标注实验性质的能力。
- 示例代码必须能指向当前 API，不引用已废弃构造函数或不存在的配置字段。
- `NotImplementedException` 如果是设计性限制，必须在异常消息或文档中说明替代入口。
- Demo/Mock 能力必须标注用途，不能描述为生产功能。
- 新增协议进入 README 前，应至少具备解析器、模板、最小测试和接入说明。
