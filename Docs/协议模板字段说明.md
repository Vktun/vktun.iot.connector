# 协议模板字段说明

本文档说明协议模板 JSON 的公共字段和各协议常用字段。模板应使用 UTF-8 编码，枚举值按 C# 枚举名称书写。

## 公共字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `ProtocolId` | string | 是 | 协议模板唯一标识 |
| `ProtocolName` | string | 是 | 展示名称 |
| `Description` | string | 否 | 模板说明 |
| `ProtocolType` | string | 是 | `ModbusRtu`、`ModbusTcp`、`S7`、`IEC104`、`Custom`、`Http`、`Mqtt` 等 |
| `ProtocolVersion` | string | 否 | 模板版本，默认 `1.0.0` |
| `Vendor` | string | 否 | 厂商 |
| `DeviceModel` | string | 否 | 设备型号 |
| `TemplateSource` | string | 否 | 模板来源 |
| `ConfigVersion` | number | 否 | 配置版本号 |
| `DefinitionJson` | string/object | 推荐 | 具体协议定义，优先级高于 `ParseRules` |
| `ParseRules` | object | 兼容 | 旧模板兼容字段，新模板不推荐依赖隐式键 |
| `Points` | array | 推荐 | 点位定义 |

## 点位字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `PointName` | string | 点位名称，必须唯一且便于业务识别 |
| `Address` | string | 统一地址表达，如 `40001`、`DB1.DBW0` |
| `Offset` | number | 原始报文偏移，适用于自定义协议 |
| `Length` | number | 字节长度或字段长度 |
| `DataType` | string | `Bool`、`UInt16`、`Int32`、`Float`、`Double`、`Ascii` 等 |
| `ByteOrder` | string | 字节序，如 `BigEndian`、`LittleEndian` |
| `Unit` | string | 单位 |
| `Ratio` | number | 比例系数 |
| `OffsetValue` | number | 线性偏移量 |
| `MinValue` / `MaxValue` | number | 合理范围，用于校验或质量判断 |
| `Quality` | number | 初始质量值 |

## Modbus 字段

| 字段 | 说明 |
| --- | --- |
| `SlaveId` | RTU 从站地址，典型范围 1-247 |
| `ModbusType` | `Rtu` 或 `Tcp` |
| `RegisterType` | `Coil`、`DiscreteInput`、`InputRegister`、`HoldingRegister` |
| `Address` | 寄存器地址，可使用 0 基地址或 3/4 区地址表达 |
| `Quantity` | 读取寄存器/线圈数量 |
| `WordOrder` | 双字或浮点的字序 |

## S7 字段

| 字段 | 说明 |
| --- | --- |
| `Rack` | 机架号，不能为负数 |
| `Slot` | 槽位号，不能为负数 |
| `Area` | `DB`、`I`、`Q`、`M` 等 |
| `DbNumber` | DB 块号 |
| `Address` | 地址表达，如 `DB1.DBW0`、`M0.0` |

## 自定义协议字段

| 字段 | 说明 |
| --- | --- |
| `FrameType` | `FixedLength`、`VariableLength`、`Separator` |
| `ByteOrder` | 字节序 |
| `FrameHeader` | 帧头字节和长度 |
| `FrameLength` | 长度字段偏移、长度和计算规则 |
| `Check` / `CheckType` | `None`、`XOR`、`CRC16`、`CRC32`、`LRC`、`Sum` |
| `Points` | 按偏移解析数据点 |

## HTTP 模板建议

HTTP 本身通常不需要复杂二进制模板，设备连接字段放在 `DeviceInfo.ExtendedProperties`：

- `Url` / `RequestUri` / `EndpointUrl`
- `BaseAddress` / `BaseUrl`
- `Path` / `EndpointPath`
- `Method` / `HttpMethod`
- `ContentType`
- `Headers`

如果响应体需要解析为点位，应提供 `HttpProtocolParser` 可识别的响应结构或上层自定义解析器。

## MQTT 模板建议

MQTT 连接字段放在 `DeviceInfo.ExtendedProperties`：

- `Server` / `Host` / `BrokerHost`
- `Port` / `BrokerPort`
- `ClientId`
- `Username` / `Password`
- `UseTls`
- `Qos` / `QosLevel`
- `CleanSession`
- `SubscribeTopics` / `Topics`

主题与 payload 的业务解析应在协议模板或业务处理层中明确，不要只依赖 topic 字符串约定。
